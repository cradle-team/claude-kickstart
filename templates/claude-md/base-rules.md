# 基本ルール

## パッケージマネージャ
- **{{PKG_MANAGER}}一択。** プロジェクトCLAUDE.mdで別指定がない限り

## TypeScript
- `any`禁止。型を書け
- 日本語コメント推奨。変数名・関数名は英語
- TODOコメント: `// TODO(@user): 説明 - YYYY-MM-DD`

## Git
- コミットメッセージ: 英語、`type: description`（feat, fix, refactor, docs, test, chore）
- mainへの直push禁止。PRを通せ
- force-pushはユーザーの明示的な許可なしにやるな

## セキュリティ
- シークレットはハードコードするな。環境変数で管理
- .envはコミットするな。.env.exampleで管理
- CORSに `*` は禁止
- セキュリティに自信がない実装は勝手にやるな。聞け

## テスト
- コードを書いたらテストも書け
- テストファイルは対象と同ディレクトリに配置

## デザイン
- AI生成っぽいデザイン禁止。グラデーション多用、幾何学パターン、ストックイラスト風は全部NG
- 実在の優れたサイトを参考にしろ
