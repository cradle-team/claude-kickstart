# コンテンツ制作パイプライン例

Worker/Manager品質ゲートパターンを使ったコンテンツ制作の具体例。

## 概要

記事・広告コピー・スクリプト等を段階的に品質を上げながら制作する。
各ステージで専門エージェントが作業し、マネージャーが品質ゲートで採点する。

## ステージ構成

### Stage 1: リサーチ
- **Worker**: Research Agent
  - 競合分析、ターゲットユーザー調査、キーワード調査を実行
  - 情報源のURL、統計データ、引用を収集
- **Manager**: Research Manager
  - 採点基準（各10点満点）:
    1. 情報の網羅性: 主要な論点を全てカバーしているか
    2. 情報の正確性: ソースが信頼できるか、最新か
    3. 競合との差別化ポイント: 独自の切り口はあるか
  - 閾値: 全項目8/10以上で合格
  - 差し戻し時: 「○○の情報が不足」「△△のソースが古い」等を具体的に指摘

### Stage 2: 構成・執筆
- **Worker**: Draft Agent
  - リサーチ結果を基に構成を組み、本文を執筆
  - 見出し、導入、本文、CTA（行動喚起）を含める
- **Manager**: Draft Manager
  - 採点基準（各10点満点）:
    1. 構成の論理性: 読者が自然に読み進められるか
    2. メッセージの明確さ: 伝えたいことが1つに絞られているか
    3. ターゲットとの適合性: 想定読者に刺さる内容か
  - 閾値: 全項目8/10以上で合格

### Stage 3: 品質チェック
- **Worker**: Quality Agent
  - 文章の推敲、冗長表現の削除、インパクトの強化
  - 各文が存在する理由を持っているか検証
- **Manager**: Quality Manager
  - 採点基準（各10点満点）:
    1. インパクト: 読者の感情を動かせるか
    2. 独自性: 他にない視点や表現があるか
    3. 冗長排除: 無駄な文が1つもないか
  - 閾値: 全項目8/10以上で合格

### Stage 4: 人間が最終確認

## 実行手順

1. リーダー（あなた）がAgent toolで各Workerを順番に起動
2. Workerの成果物をManagerに渡して採点させる
3. 不合格なら差し戻し（最大3回）
4. 合格なら次のステージへ
5. 全ステージ完了後、人間が最終確認

## 注意事項

- 各ステージの中間成果物は必ずファイルに保存しろ
- 差し戻しの履歴も記録に残せ（何が弱かったか、どう改善したかの学び）
- 3回差し戻しても合格しない場合は人間にエスカレーション
